# --- Consigna ---

Realizar una conexión local con implicancias de conexión de red con un proceso de servicio hacia una de las puntas

# --- Descripcion ---
Primero, se realiza una conexión local desde la computadora del usuario hacia un repositorio en GitHub.
Para esto, es necesario tener Git instalado y configurar el acceso con usuario, correo y autenticación mediante contraseña o clave SSH.
La conexión comienza cuando el desarrollador crea una carpeta de proyecto y la vincula con GitHub usando comandos como git init, git remote add origin y git push.
Así, el repositorio local queda conectado con el repositorio remoto.
La conexión de red permite enviar y recibir archivos entre la computadora y los servidores de GitHub mediante protocolos seguros como HTTPS o SSH.
GitHub actúa como proveedor del servicio, ya que recibe los cambios enviados, los guarda en la nube y permite que otros usuarios puedan acceder, descargar o modificar el proyecto según los permisos asignados.
